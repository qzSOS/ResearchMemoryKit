# 理论说明

本文档总结 ResearchMemoryKit 的公开版设计理论。

## 系统目标

1. **可恢复**：新会话能快速恢复到足够继续工作的上下文。
2. **可追溯**：决策、结果和失败都有来源。
3. **可信决策**：文件中的信息可靠到足以指导下一步。
4. **方向纪律**：项目转向、停止和晋级通过明确门控，而不是依赖惯性。
5. **可复现工程**：命令、元数据、输出和结论保持连接。
6. **节省人的认知带宽**：系统应该减少维护负担，而不是制造新的负担。

## 公理

### A1：知识价值只能事后判断

一条笔记在创建时往往无法判断价值。一次 debug 观察可能后来成为关键结论，一个小环境问题也可能解释整个失败分支。

设计含义：不要要求所有知识在创建时就被完美分类。

### A2：科研工作流熵增是正常现象

长期项目一定会产生失败路线、旧输出、临时脚本和半退役约定。这不是管理失败，而是探索过程本身。

设计含义：追求优雅降级，而不是追求完美整洁。

### A3：上下文必须分层

人的注意力和模型上下文都有限。单一大文件最终会变得不可读。

设计含义：使用 Router + Leaf 结构：入口短，细节按需读取。

### A4：只追加能保存可信度

静默覆盖会摧毁信任。如果旧结果是错的，保留旧记录并追加修正。

设计含义：决策、失败和结论应当只追加。

### A5：稀缺资源会压缩理想流程

GPU 时间窗、截止日期、远程环境、交付压力都会把人推向最短路径。

设计含义：让降级路径成为默认路径。只在从容条件下有效的规则一定会被绕过。

### A6：方向判断仍然属于人

Agent 擅长结构化执行、比较文件、运行验证和维护记录。但“什么值得做”仍然需要人判断。

设计含义：记忆层服务人的判断，而不是假装替代人的判断。

## 失败模式

### FM-1：知识漂移

长期文件逐渐偏离真实项目状态。

缓解：Current State 加完成门。

### FM-2：静默覆盖

旧信息被替换，但没有修正轨迹。

缓解：决策、失败、结论只追加。

### FM-3：记忆沉积

临时笔记一直留在暂存区，恢复上下文越来越慢。

缓解：短小的可覆盖快照，配合定期迁移到稳定记录。

### FM-4：真源重复

同一个可变事实出现在多个文件里。事实变化时，只有部分副本被更新。

缓解：一个可变事实只保留一个权威来源；Router 只链接，不复述。

### FM-5：输出墓地

生成产物不断堆积，既舍不得删，又找不到可信用途。

缓解：把输出分类为已接受证据、诊断产物、归档或外部存储。

### FM-6：孤儿决策

决策没有理由、备选方案或复盘条件。

缓解：每条决策都记录问题、理由、备选方案和复盘条件。

### FM-7：草稿和记录混淆

实验日志同时承担保留结论和清晰展示两种职责。对整个文件机械执行只追加，会制造重复真源。

缓解：结论只追加，展示格式可以在不删除结论的前提下重排。

## 设计原则

### P1：当前状态可覆盖，历史记录只追加

当前事实用可覆盖文件，历史来源用只追加文件。

### P2：Day-0 模板仍然需要完成门

文件存在不等于行为激活。只有把“记忆已更新”纳入完成定义，Agent 才会稳定维护它。

### P3：Router 不应承载会变化的项目事实

Router 只放恢复顺序、文件地图、稳定规则和链接。当前结论属于 Current State。

### P4：失败路线是一等证据

失败减少未来搜索空间，不是应该隐藏的脏东西。

### P5：来源比完整更重要

一个带 commit、配置、输入角色和状态的简短记录，比永远写不完的完美记录更有价值。

### P6：只追加日志变慢时加索引

只追加文件会变大。检索变慢时，应增加活动索引或摘要，而不是删除历史。

### P7：记忆层要门控进展，而不只是描述进展

记忆层应定义什么算完成实验、接受结论、有效转向或合格交付。这样 Agent 可以自主执行，但项目进展仍然可审计。

### P8：可信来自闭环

Agent 可以运行很多命令，却没有真正推进科研。只有决策、执行、验证、结论或失败、坑位更新和 Current State 替换全部闭环，进展才可信。

### P9：显式契约可以让门控接受机器检查

Markdown 记录和人类可读门控定义工作流。需要可靠自动化的项目，可以再增加一个
小型机器可读契约，明确权威文件、路由、状态新鲜度和门控。

设计含义：契约应保持显式、可检查，同时检查器不能膨胀成通用工作流引擎。

书面门控和自动检查承担不同职责。书面门控定义项目在接受任务前需要哪些
证据和记录；P0 只验证声明的文件、路由、状态字段和门控标题是否仍然存在，
不会判断证据是否满足门控。

## 它不是什么

- 不是 ML 实验追踪平台的替代品；
- 不是向量数据库；
- 不是完整 Agent 操作系统；
- 不是项目管理工具；
- 不能保证没有完成门的 Agent 会自动维护记忆。
